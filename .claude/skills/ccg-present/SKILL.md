# CCG 발표 전략 가이드

1. `README.md` — 제품 정의·핵심 차별점·주요 기능 (서사의 뼈대)
2. `docs/REASONING_ARCHITECTURE.md` — 룰(결정론)/LLM(해석) 이원 구조 다이어그램
3. `체크리스트.md` — 대회 요구사항 체크 상태
4. `docs/CHANGELOG_FEATURES.md` — 기능 이력
5. `docs/[데이콘] JB금융그룹 Fin AI Challenge MVP제안서_제출본_JunBub 팀.pdf` — 제출본
   제안서 (기존 서사와 일관성 유지, PDF는 pages 파라미터로 나눠 읽기)
6. `판정5종_설명_슬라이드.png` — 기존 판정 5종 설명 시각자료 (재활용 후보)
7. 리더가 전달하는 최신 실측·평가 산출물 (`_workspace*/` 요약, `eval/violation_sweep/`)

## 서사 원칙

- **문제 → 구조 → 증거 순서**: 금융광고 사전심사의 실무 고통(수작업·비일관·규제
  리스크) → 룰/LLM 역할 분리 구조가 왜 신뢰를 만드는가 → 실데이터 검증 결과.
- **차별점 3축에 종속시켜 배치**:
  1. 설명가능성 — 모든 판정이 근거 조문 원문까지 추적 (금감원 회답식: 정의→요건→결론→유보)
  2. 신뢰 계약 — 조용한 fallback 없음, 데이터 기반 필수고지 카탈로그, 감사 추적
  3. 실전성 — 실은행 상품 데이터 스윕, 다국어(KR/KH 2-DB), 운영 대시보드
- **숫자는 출처와 함께**: "5분→구간 35% 단축(실측)", "실상품 N건 중 위반 M건 검출"
  처럼 산출물 파일을 출처로 명시. 파일럿 진행 중이면 자리표시자로 두고 표시.
- 기능 나열식 목차 금지 — 각 목차는 심사위원이 얻어갈 결론 1문장이 제목 뒤에 붙어야 한다.

## 산출물 형식

목차 항목마다:
- **핵심 메시지** (1줄 — 이 슬라이드에서 심사위원이 기억할 것)
- **내용 요점** (3~5 불릿)
- **근거/출처** (파일 경로·수치)
- **시각자료/데모 제안** (기존 자산 재활용 우선: 콘솔 3-pane 라이브, 판정5종 이미지 등)
- **예상 질문 + 답변 방향**
- **시간 배분**

마지막에 전체 리허설 체크리스트와 리스크(까다로운 질문 Top 5)를 붙인다.
