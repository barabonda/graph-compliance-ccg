# JB Compliance 발표 자료 브리프

## 서비스 요약

서비스명:

- 기본: `JunBub`
- 보조 표기: `JB Compliance · Content Safeguard`

주제:

- 지정주제 2. Compliance AI

한 줄 소개:

> 금융광고 초안을 상품설명서 사실, 금소법/심의기준, 필수 고지와 대조해 문구별 리스크·근거·수정안을 제시하는 준법 사전심사 AI Agent.

핵심 가치:

- 마케팅 담당자는 어디를 고쳐야 하는지 빠르게 안다.
- 준법 담당자는 왜 그런 판단이 나왔는지 근거를 본다.
- 조직은 광고 배포 전 위반 가능성과 누락 고지를 줄인다.

## 발표의 큰 흐름

Track A/B를 따로 기능처럼 설명하지 않는다. 전체 심사 방법 안에 두 판단 축이 포함되어 있다고 설명한다.

추천 표현:

```text
1. 먼저 광고 문구를 문장과 주장 단위로 구조화한다.
2. 상품군/채널/행위요건에 맞지 않는 기준은 제외한다.
3. 적용되는 기준만 상품문서, 필수고지, 법령 근거와 대조한다.
4. LLM은 정리된 근거 창 안에서 개별 조항 판단과 전체 인상 판단을 수행한다.
5. 결과는 수정안과 감사 가능한 근거 경로로 남긴다.
```

## 2. Problem Definition

목표:

- "왜 이 문제가 중요하고 지금 방식으로는 어려운가"를 설득한다.

메시지:

- 금융광고 심사는 문구만 보는 작업이 아니다.
- 상품 조건, 필수 고지, 법령/심의기준, 소비자가 받는 전체 인상을 같이 봐야 한다.
- 수작업 검토는 시간이 오래 걸리고 누락 위험이 있다.

예시 문구:

```text
누구나 조건 없이 최고 연 5.0% 확정 금리
```

이 문구 하나에 필요한 판단:

- 누구나 가입 가능한가?
- 최고금리는 기본금리인가 우대금리인가?
- 우대조건과 가입기간이 고지되었는가?
- 예금자보호 한도는 정확한가?
- 소비자가 확정 수익으로 오인하지 않는가?

추천 그림:

- 광고 문구 하나에서 여러 확인 항목으로 갈라지는 문제 도식

## 3. Solution Overview

목표:

- 전체 시스템을 한 장으로 설명한다.

핵심 구조:

```text
광고 초안
→ Context Graph
→ Policy Graph
→ Product Fact Graph
→ Disclosure Gate
→ LLM Judge
→ 수정안 / 근거 / Audit
```

쉽게 설명:

- Context Graph: 광고 문구를 claim, qualifier, consumer effect로 쪼갠다.
- Policy Graph: 금소법, 시행령, 감독규정, 심의기준, CU를 연결한다.
- Product Fact Graph: 상품설명서에서 금리, 조건, 기간, 보호한도 같은 사실을 뽑는다.
- Disclosure Gate: 지금 상품과 채널에 필요한 고지만 검사한다.
- LLM Judge: 구조화된 근거 안에서 위반 가능성과 수정안을 판단한다.

추천 그림:

- 5단계 파이프라인 도식
- 기존 RAG와 차이를 보여주는 비교 그림

## 4. Key Features

목표:

- MVP에서 실제로 제공하는 기능을 업무 액션 중심으로 보여준다.

기능 1. 새 심사와 대기열

- 여러 광고 초안을 넣고 순차 처리한다.
- 실행 조건과 모델 설정을 기록해 같은 조건으로 재심사할 수 있다.

기능 2. 문구별 리스크

- 광고 원문에서 위험 표현과 유지해야 할 고지를 하이라이트한다.
- `누구나`, `조건 없이`, `확정`, `보장` 같은 표현은 parent claim 안에서 판단한다.

기능 3. 상품 사실 대조

- 선택한 상품의 상품설명서/주요내용/약관에서 ProductFact를 추출한다.
- 광고 ClaimFact와 비교해 `SUPPORTED`, `CONTRADICTED`, `CONDITION_MISSING`, `NOT_TESTED`를 구분한다.

기능 4. 필수 고지와 현저성

- 예금자보호, 우대조건, 가입기간, 세전/세후, 상품설명서 확인 등 필요한 고지를 검사한다.
- 고지가 있어도 혜택 문구보다 너무 약하게 보이면 `PROMINENCE_INSUFFICIENT`로 표시한다.

기능 5. 근거와 수정안

- 금소법/시행령/감독규정/심의기준 근거를 보여준다.
- Before/After 수정안을 제공하고, 최종 판단은 준법 담당자가 한다.

추천 그림:

- 실제 콘솔 캡처 2장:
  - 문구별 리스크 화면
  - 수정안 Before/After 화면

## 5. Data & Tech

목표:

- 구현 가능성과 기술적 깊이를 보여준다.

데이터:

- 금융광고 심의 기준/세칙
- 금소법, 시행령, 감독규정, 표시광고 관련 기준
- JB 상품문서 메타데이터
- 상품설명서/상품주요내용/약관 PDF
- 테스트용 금융광고 문안과 위반 유형 라벨

기술:

- Backend: FastAPI 기반 review workflow
- Frontend: Next.js/React 콘솔
- Graph: Neo4j Policy/Product/Context Graph
- Retrieval: graph retrieval, embedding, cross-encoder rerank
- LLM: OpenAI/Claude/local model 교체 가능 구조
- Evaluation: article-level multi-label metrics + CCG diagnostics

핵심 구현 포인트:

- LLM에게 전체 법령을 던지지 않고, 적용되는 기준과 evidence window만 전달한다.
- 적용범위 Gate가 안 봐도 되는 기준을 OFF 처리해 오탐을 줄인다.
- ProductFact가 없으면 사실처럼 만들지 않고 `NOT_TESTED` 또는 `NEEDS_PRODUCT_SELECTION`으로 둔다.

추천 그림:

- 기술 아키텍처 다이어그램
- ProductFact comparison mini table

## 6. User Scenario

목표:

- 심사위원이 실제 사용 장면을 상상하게 한다.

사용자:

- 마케팅 담당자: 광고 초안을 등록하고 수정안을 반영한다.
- 준법 담당자: 위험 문구, 근거 조항, 필수 고지, 수정안을 검토한다.
- 팀장/관리자: 반복 위험, 처리 현황, 감사 기록을 본다.

흐름:

1. 마케팅 담당자가 광고 문안을 등록한다.
2. 상품명을 검색/선택한다.
3. Agent가 광고를 문구별로 분해하고 상품 사실과 고지를 대조한다.
4. 준법 담당자가 위험 카드와 근거를 확인한다.
5. 수정안을 적용하거나 보완 요청/반려/승인한다.
6. 실행 조건, 근거, 판단 이력이 저장된다.

추천 그림:

- 6단계 사용자 여정
- "마케터 → Agent → 준법 담당자" 업무 흐름

## 7. Expected Impact

목표:

- 이 서비스가 JB금융그룹에 어떤 변화를 줄 수 있는지 말한다.

기대 효과:

- 광고 사전 검토 시간 단축
- 필수 고지 누락 감소
- 상품 설명서와 광고 문구의 불일치 조기 발견
- 준법 담당자의 반복 검토 부담 완화
- 근거 중심의 커뮤니케이션으로 마케팅/준법 협업 개선
- 감사 대응 가능한 판단 이력 축적

확장성:

- 예금/적금에서 대출, 투자, 보험, 카드 광고로 확장
- 웹페이지에서 SNS, 배너, 유튜브 자막으로 확장
- JB금융 계열사별 내부 심의 기준 연결
- 규정 변경 모니터링과 자동 회귀 테스트로 운영 고도화

마무리 문장:

> JunBub은 광고를 막는 AI가 아니라, 더 빠르고 안전하게 고객에게 나갈 수 있도록 근거와 수정 방향을 제공하는 준법 파트너입니다.
