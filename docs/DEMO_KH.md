## ★ 시연 전 체크리스트

1. **기동 + 워밍업**
   - 백엔드: `cd repo && source ../.venv/bin/activate && uvicorn server:app --port 8770`
   - 프론트: `cd repo/frontend && npm run dev` (:3000)
   - `curl http://localhost:8770/health` → `{"status":"ok"}`
   - **워밍업 심사 1회 필수**: Aura 유휴 커넥션 드롭으로 첫 심사가 500으로 실패한 이력 있음(`OSError: No data`). 짧은 광고 1건을 미리 심사해 커넥션을 깨워둘 것. 본 시연 중 500이 나오면 그냥 **재실행**(재시도로 항상 성공했음).
2. **.env KH 오버라이드 확인** (`repo/.env`)
   - `OPENAI_API_KEY` 유효(빈 줄/중복 줄 금지 — env_loader는 first-wins)
   - `CCG_MIN_HYPERNYM_graphcompliance_cambodia_ppcbank_20260630=10` / `CCG_MIN_PREMISE_...=15`
   - `CCG_NON_KR_LAW_WORKSPACES=graphcompliance_cambodia_ppcbank_20260630`
   - `CCG_PRELOADED_PRODUCT_FACTS_WORKSPACES=graphcompliance_cambodia_ppcbank_20260630`
   - `WORKSPACE_ID=graphcompliance_cambodia_ppcbank_20260630` (상품 검색용)
   - `JB_PRODUCT_DISCLOSURE_ROOT=<repo>/data/cambodia/products`
3. **판정문 문체 확인**: 판정 근거에 한국 프롬프트 문체 echo("법률→시행령→감독규정→심의기준 위임사슬에 준하는…")가 보이면 **재실행**. 확률적으로 발생(인용 조문 자체는 항상 캄보디아 법이지만, 시연 화면에선 지저분해 보일 수 있음).
4. 시연용 광고 텍스트는 `data/cambodia/eval/kh_eval_v0.json`에서 복사(위반 2종 + 깨끗한 실광고 2종).

## ★ 제약 (질문 대비)

- **KR 데이터는 팀원 공용 Aura에, KH는 내 Aura에 분산** — 분리 라우팅으로 해결: **코퍼스 읽기(retriever/legal_hierarchy/챗 도구)는 팀 DB로, 심사 산출물 쓰기(persistence/스냅샷)는 내 Aura로**. 팀 합의("새 노드/엣지는 상호 합의 후에만") 준수 — KR 라이브 심사 전후 팀 DB 노드/엣지 수 불변 검증됨(77,702/151,908). **⑥단계 KR 라이브 심사 실행 가능**(2026-07-02 검증: 금소법 시행령 제20조·감독규정 제19조 인용, reject). 단 KR **상품 대조**는 여전히 제한(상품 검색이 env 기반 KH 고정 — 법령 축만 라이브).
- **상품 검색은 env(WORKSPACE_ID) 기반**: 상품 대조 데모는 KH로 설정된 이 백엔드에서만 동작. KR·KH를 **한 백엔드에서 동시에** 상품 대조까지 시연하는 것은 불가(법령 축 심사는 payload workspace_id로 정상 분리됨). per-review 스레딩은 후속 과제.
- **NBC 금리상한(KH-CU-05)은 citation-only placeholder**: NBC 사이트가 봇 차단(403)이라 원문 미확보. **대출 금리상한을 데모 주제로 삼지 말 것.** (자동차대출 광고 데모도 이 축은 피함)
- **PoC 한계**: CU 16건·상품사실 7건은 캄보디아 법률 전문가/PPCBank 검증 전. 노드에 `poc=true, lawyer_verified=false` 명시. "검증된 사실만 페이지 원문에서 발췌" 원칙(PPCBank 페이지에 없는 최소금액·중도해지율은 **의도적으로 미적재**).
- 정기예금 상품문서는 PDF가 아닌 **웹페이지 발췌(.md)** — 문서 뷰어(`/api/product-doc`)는 PDF 전용이라 원문 뷰어 클릭은 데모에서 생략.
