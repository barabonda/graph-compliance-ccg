# 작업 로그 — 2026-06-14 (Team JunBub · JB Fin AI Challenge)

JB Compliance · CONTENT SAFEGUARD 하루치 작업 정리.

## 한 줄 요약

심사 콘솔의 판정·근거 표현을 준법 도메인에 맞게 정리하고, 로컬 LLM(Ollama)
연결과 모델 선택, 운영 대시보드(실행 기록·디버깅), 새 심사 화면·브랜딩을 추가했다.

## 추론/판정 품질

- **2층 심사 구조**: 위험 카드를 '개별 심사(표현·고지를 조항별로)'와 '종합 심사
  (전체 인상)'로 분리. 개발 용어 Track A/B를 화면에서 제거.
- **종합 심사(Track B) 그래프**: 컨텍스트 디버그 탭을 폐지하고, 혜택 주장 중심으로
  완화/강화 문장을 잇는 세로 그래프 + 종합 결론으로 Track B 카드 펼침에 통합.
- **judge가 한정 문장을 함께 본다**: 혜택 주장을 QUALIFIES/MITIGATES로 한정하는
  다른 문장을 evidence window에 넣어, 조건이 별도 문장에 있을 때 단정 과판정을 방지.
- **개발 용어 정리**: 판정 텍스트의 EvidenceWindow·INSUFFICIENT 등 내부 용어를
  표시 직전 도메인 용어로 치환 + judge 프롬프트에 금지 규칙.

## 상품 사실 · 필수 고지

- **상품문서 대조**: 누락 필수 고지가 상품설명서(product_facts)에 있는지 교차
  확인('광고엔 없지만 문서엔 있음' vs '둘 다 없음').
- **데이터 기반 카탈로그**: 어떤 고지가 어느 상품군에 필요한가를 Neo4j
  `DisclosureRequirement`(disc_*) 카탈로그에서 가져온다(코드 하드코딩 대체).

## 수정안 / 하이라이트

- **교정본 재설계**: per-span 짜깁기를 폐기하고 광고 전체를 일관되게 재작성한
  완본을 원문↔교정본 diff로. 제안(교체 문안)과 심사자 조언 분리.
- **하이라이트**: 단어별 파란 패치워크를 없애고 claim 단위 한 span으로. 위험은
  심각도 색 텍스트 + 물결 밑줄, 고지는 초록 밑줄, 선택은 같은 tone 배경(파랑 X).
- **원칙 태그 색**: 부당권유=빨강 / 광고규제=파랑 / 설명의무=보라.

## 운영 / 로컬 LLM / 배포

- **로컬 LLM 토글**: 클라우드(OpenAI Responses)와 로컬(Ollama Chat Completions +
  json_schema 강제)을 .env로 전환. 모델 드롭다운에서 요청별 모델 선택, 로컬 태그면
  로컬 엔드포인트로 라우팅(클라우드 기본 모드여도). 5개 로컬 모델 동작 확인.
- **운영 대시보드**: 실행 시점 ReviewOutput 스냅샷 저장(runs/) + 집계(라우팅 분포·
  자주 걸리는 원칙/고지/CU) + 실행 기록 표. 행 클릭 시 그 실행의 시점 데이터를
  콘솔에서 디버깅. 감사 추적 통합(별도 탭 폐지). 데모 기록은 집계에서 제외.
- **실행자 가명**: 인증 없이 브라우저별 가명(닉네임 풀)을 부여해 누가 실행했는지 표시.
- **새 심사 화면**: 샘플 프리셋·내부식별 제목·채널별 형식 요건·문안 글자수·상품
  미선택 경고·판정 모델 선택 + 친절한 분석 흐름 설명.
- **브랜딩**: 서비스명 **JB Compliance**, 팀 **JunBub**, 포지셔닝 **CONTENT SAFEGUARD**.
